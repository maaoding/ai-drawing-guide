# 工作流和模板区别

一句话：工作流是“怎么执行”，模板是“从哪里开始复用”。

## 工作流是什么

工作流是一套生成流程。它规定：

- 加载哪些模型。
- 提示词怎么编码。
- latent 怎么生成或导入。
- 使用什么采样器。
- 是否使用 LoRA、ControlNet、参考图。
- 最后如何解码和保存图片。

在 ComfyUI 中，工作流通常是一张节点图。节点之间的连线决定数据怎么流动。

## 模板是什么

模板是可复用的起点。它可能是：

- 一个已经搭好的工作流。
- 一组默认参数。
- 一个页面布局。
- 一个提示词结构。
- 一个项目文件夹结构。

模板的重点是“复用”，不是“执行逻辑”本身。

## 两者关系

一个工作流可以作为模板保存。一个模板也可以包含工作流。

例如：

- 文生图工作流：负责从文字生成图片。
- 文生图模板：预先设置好模型、尺寸、常用负向提示词和保存路径。

## 初学者应该怎么用

建议顺序：

1. 先用最简单的文生图工作流。
2. 看懂每个节点负责什么。
3. 保存一份自己的默认模板。
4. 再扩展图生图、局部重绘、放大、参考图控制。

## 常见工作流类型

- 文生图：文字生成图片。
- 图生图：基于已有图片重新生成。
- Inpaint：局部重绘。
- Outpaint：向画面外扩展。
- Upscale：放大和细节修复。
- ControlNet：通过姿势、线稿、深度图控制结构。
- IP-Adapter：使用参考图影响风格或人物特征。

## 常见误区

- 以为导入工作流就一定能跑。工作流依赖模型、节点和文件路径。
- 把工作流当成模型。工作流没有生成能力，模型才有。
- 不看缺失节点和缺失模型提示。报错通常已经指出缺什么。
- 一次改太多节点。调试时应该固定变量。
