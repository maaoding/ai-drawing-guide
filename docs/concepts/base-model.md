# 底模是什么

一句话：底模是 AI 绘画系统的核心生成模型，决定它“会画什么、擅长什么、默认风格是什么”。

## 底模、Checkpoint、Base Model 的关系

在不同教程里，你会看到几个相近的词：

- 底模：中文习惯说法，指生成图片的核心模型。
- Checkpoint：常见文件形式，通常是 `.safetensors` 或 `.ckpt`。
- Base Model：训练或微调时的基础模型，也常用于说明 LoRA 兼容性。

实际使用时，很多人说“换底模”，就是在 WebUI 或 ComfyUI 里换一个 checkpoint。

## 为什么底模很重要

底模会影响：

- 默认画风：写实、动漫、插画、设计图、摄影感。
- 对提示词的理解方式：自然语言、标签、混合写法。
- 支持的分辨率：例如 SD1.5 常见 512 系，SDXL 常见 1024 系。
- 可用 LoRA：LoRA 通常只能在相近生态里稳定工作。
- 图片细节和稳定性：人体、手、文字、背景复杂度都会受影响。

提示词不能完全替代底模。一个偏写实的模型很难只靠提示词稳定变成纯动画截图风格；一个偏二次元的模型也未必适合真实产品摄影。

## 常见底模生态

### SD1.5

早期 Stable Diffusion 生态。优点是资源多、显存要求低、插件和教程多。缺点是基础分辨率和画面细节较旧。

### SDXL

比 SD1.5 更新的一代 Stable Diffusion 生态。更适合 1024 分辨率，画面结构和细节通常更好。

### Pony

基于 SDXL 生态发展出的二次元/角色向模型体系。常见于角色、动漫、标签式提示词和大量 LoRA 玩法。

### Illustrious / NoobAI

偏二次元插画与标签体系的模型生态。资源和 LoRA 多，但不要默认与 Pony 完全兼容。

### FLUX

新一代图像生成模型生态，通常更擅长自然语言理解、质感和复杂画面。但工作流、模型结构和 LoRA 兼容性与 SDXL 系不同。

### Qwen-Image / Z-Image

偏新一代多模态或图像生成体系，常见特点是语言理解、中文能力、文字渲染或图像编辑能力更强。生态还在快速变化，使用时要看模型卡和工作流说明。

## 如何判断 LoRA 是否适合某个底模

优先看模型页面的 `Base Model` 或说明：

- 写 SD1.5 的 LoRA，不要默认给 SDXL 用。
- 写 SDXL 的 LoRA，不要默认给 FLUX 或 Qwen-Image 用。
- 写 Pony / PonyXL 的 LoRA，优先配 Pony 系模型。
- 写 FLUX.1 的 LoRA，不要默认配 FLUX.2。
- 写 Z-Image 的 LoRA，优先配 Z-Image 系模型。

判断不清时，先低权重测试，例如 0.3 到 0.6。

## 常见误区

- “模型越新越好”：不一定。生态、LoRA、教程和显存需求同样重要。
- “所有 `.safetensors` 都一样”：不一样。它可能是 checkpoint、LoRA、VAE 或文本编码器。
- “只要名字像就能混用”：不可靠。要看模型结构和训练说明。
