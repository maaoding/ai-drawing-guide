入门路线 ​

AI 绘画入门最容易卡在两个地方：一是术语太多，二是教程默认你已经知道模型和工作流怎么配。建议按下面顺序学。

第一步：理解“模型负责什么” ​

一句话：模型决定图片大方向，提示词和参数只是引导它。

先弄清楚这些词：

  • 底模：负责生成能力的核心模型。
  • LoRA：给底模临时追加某种角色、画风或概念的小模型。
  • VAE：负责把模型内部的潜空间结果转换成可见图片。
  • 文本编码器：把提示词转成模型能理解的向量。

如果这些词没区分开，后面很容易把“不兼容”“没加载”“提示词写错”混成一个问题。

第二步：学会最小出图流程 ​

一个最小的文生图流程通常包括：

  1. 选择底模。
  2. 写正向提示词。
  3. 写负向提示词。
  4. 设置图片尺寸、步数、CFG、采样器。
  5. 运行生成并保存图片。

先不要急着叠多个 LoRA、ControlNet、高清修复。初学阶段先追求“知道每个改动会造成什么影响”。

第三步：学会判断模型生态 ​

不同模型不是简单的新旧关系，而是生态不同。

  • SD1.5：老生态，资源多，显存要求低，但基础画质和分辨率较旧。
  • SDXL：更适合高分辨率和通用画质。
  • Pony / Illustrious / NoobAI：偏二次元和标签体系，LoRA 生态强。
  • FLUX / Qwen-Image / Z-Image：新一代模型，文字理解和质感更强，但工作流与 LoRA 生态不一定通用。

第四步：建立自己的工作流 ​

工作流不是玄学配方，而是可复用的节点流程。你应该逐步建立几类基础工作流：

  • 文生图：从文字生成图片。
  • 图生图：在原图基础上改风格、改细节。
  • 局部重绘：只改图片某个区域。
  • 放大修复：提高分辨率并补细节。
  • 参考图控制：用参考图影响人物、姿势、风格或构图。

第五步：再进入进阶主题 ​

当你能稳定理解“为什么这次出图失败”后，再学习：

  • LoRA 训练。
  • ControlNet 和姿势控制。
  • IP-Adapter / 参考图一致性。
  • 高清修复和分块放大。
  • ComfyUI 自定义节点。
  • 批量生成和自动化。

常见误区 ​

  • 只换提示词，不换模型。很多风格问题根本不是提示词能解决的。
  • 乱下 LoRA。LoRA 必须看训练底模，不能假设通用。
  • 一开始就堆复杂工作流。节点越多，越难定位问题。
  • 忽略许可证。公开发布作品前，应确认模型、LoRA、素材和平台规则。
